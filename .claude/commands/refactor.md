# Refactor Code

Refactor the requested code while preserving behavior and architecture consistency.

Goals:

* improve readability
* improve maintainability
* reduce duplication
* simplify logic
* improve scalability
* improve developer experience

Requirements:

* preserve existing functionality
* avoid unnecessary rewrites
* avoid introducing overengineering
* preserve API contracts unless explicitly requested
* maintain Docker compatibility
* maintain typing correctness

Focus on:

* reusable abstractions
* smaller focused functions
* cleaner naming
* separation of concerns
* reducing nested logic
* removing duplication
* improving file organization

Backend refactoring priorities:

* thin routes/controllers
* better service separation
* cleaner repository usage
* improved async flow
* improved dependency management

Frontend refactoring priorities:

* reusable UI components
* reusable hooks
* cleaner state management
* reducing prop drilling
* separating UI from logic

After refactoring:

1. explain improvements made
2. explain tradeoffs
3. identify future refactoring opportunities
4. provide a professional git commit message

# Review Code

Review the current codebase changes like a principal engineer.

Focus areas:

* architecture consistency
* scalability
* maintainability
* readability
* performance
* security
* developer experience
* Docker compatibility
* frontend/backend integration quality

Review backend for:

* service separation
* repository usage
* async correctness
* validation quality
* API consistency
* database query efficiency
* logging and error handling

Review frontend for:

* component reusability
* state management quality
* separation of concerns
* rendering performance
* loading/error handling
* accessibility
* responsive behavior

Identify:

1. critical issues
2. architectural concerns
3. maintainability risks
4. performance concerns
5. unnecessary complexity
6. opportunities for simplification

Do not rewrite the entire codebase unnecessarily.

Prefer pragmatic improvements.

At the end:

* provide production-readiness assessment
* provide scalability assessment
* suggest a professional commit message

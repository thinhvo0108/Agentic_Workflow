# Implement Feature

Implement the requested feature following all architecture and engineering rules defined in CLAUDE.md.

Requirements:

* preserve architecture consistency
* maintain clean separation of concerns
* avoid unrelated refactors
* generate production-grade code
* include validation
* include error handling
* include proper typing
* include loading/error states where applicable
* ensure Docker compatibility

Before coding:

1. explain the implementation plan
2. identify affected files
3. explain architectural decisions
4. mention scalability considerations

Implementation requirements:

* avoid placeholder pseudocode
* avoid fake TODO implementations
* prefer reusable abstractions
* keep functions/components focused
* preserve readability

Backend expectations:

* thin controllers/routes
* business logic in services
* repository pattern where appropriate
* async implementation
* structured logging

Frontend expectations:

* reusable components
* TypeScript strict typing
* responsive UI
* proper loading/error states
* clean state management

After implementation:

1. summarize changes
2. mention tradeoffs
3. suggest improvements if applicable
4. provide a professional git commit message
